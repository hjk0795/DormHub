# Planner Ops Flow Check

Use this skill before proposing product tasks for a dorm operations problem.

Your job is to identify the real operational bottleneck first, then recommend only the smallest useful next tasks.

## Use This Skill When

- The user describes a resident problem, admin workload, or repeated dorm inquiry
- The planner is about to suggest product tasks
- The issue may involve multilingual confusion, self-service failure, vague reporting, or repeated staff explanation

## Internal Check

Before proposing any task, check:

1. What is the real recurring operational problem?
2. Who is affected: resident, admin, or both?
3. Where is the friction in the resident/admin flow?
4. Does this create repeated questions, repeated explanations, or manual admin follow-up?
5. Can this be solved with a smaller change instead of a larger feature?
6. Does this improve operations now, not add speculative future functionality?

## Planning Rules

- Keep scope small
- Prioritize real operational impact
- Prefer clearer flow, guidance, structure, or intake over new feature expansion
- Avoid implementation details unless the user explicitly asks for them
- Avoid speculative redesigns
- Balance resident convenience with admin efficiency

## Output

Return:

1. Problem summary
2. Ops Flow findings
3. Up to 3 recommended small tasks
4. Best single recommendation with reason

## Quality Bar

- If one task can solve most of the pain, recommend one task
- If the problem is mostly copy, structure, or flow clarity, do not invent a large feature
- If the issue comes from inconsistent handling, favor standardization and structured intake
- If the issue is unclear, separate the surface complaint from the operational root cause
